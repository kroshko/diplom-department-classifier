# Прогнозирование выбора кафедры студентами ИТММ

## Описание задачи

Разработка модели машинного обучения для прогнозирования распределения студентов по кафедрам на основе их академических показателей за 2 года обучения.

**Данные:** 250 студентов, 2 приоритета выбора кафедры.

**Целевая переменная:** Группа кафедр (3 группы):
- Группа 1: ДУМЧА, АГДМ — математические направления
- Группа 2: МОСТ
- Группа 3: ТУДС, ТВИАД, ТКЭМ, ПМ, ИАНИ — смешанные направления

**Признаки:**
- Баллы ЕГЭ (Русский язык, Математика, Информатика, Физика)
- Средний балл (`sr_o`)
- Индивидуальные достижения
- Направление подготовки

---

## Предобработка данных

1. **Категориальные признаки** → числовые (кафедры, направление подготовки)
2. **Пропуски:** `Информатика`, `Физика` заполнены нулями
3. **Удаление:** строки с пропусками в целевых признаках
4. **Стандартизация:** `StandardScaler` — обучен на train, применён к test

---

## Подход к обучению

### Корректный пайплайн (без утечки данных):

```
train_test_split (stratify=y)
    → StandardScaler.fit_transform(X_train)
    → StandardScaler.transform(X_test)
    → Обучение модели на X_train_scaled
    → Оценка на X_test_scaled
```

## Балансировка классов

### Первый приоритет
Распределение классов: ~14% / 40% / 46% — умеренный дисбаланс.
SMOTE не применялся, использован `class_weight='balanced'` в моделях.

### Последний приоритет
Распределение классов: ~50% / 4% / 46% — критический дисбаланс.
Группа 2 представлена всего 11 студентами (4.4% от выборки).
При стратифицированном разбиении в тестовую выборку попало только 2 студента
группы 2, что делает любые предсказания для этой группы статистически недостоверными.

---

## Модели

Сравнены 6 моделей:
1. Логистическая регрессия
2. Дерево решений
3. Случайный лес
4. Градиентный бустинг
5. Сбалансированный случайный лес
6. Бэггинг-классификатор

**Кросс-валидация:** StratifiedKFold (5 folds) для всех моделей

---
